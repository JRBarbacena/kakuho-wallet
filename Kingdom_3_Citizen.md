# Kingdom 3: The Driver (Prover)

The citizen (or driver) is the owner of their data. They interact with the system via the `web-citizen` application.

## 📱 Application: `web-citizen`

### The Digital Wallet (`/wallet`)
- **Storage**: Private data is stored in the browser's `localStorage`.
- **Fields**: `secret`, `licenseID`, `fullName`, `leafIndex`.
- **Security**: The data never leaves the client in its raw form, except when the user chooses to "show" their public name during a proof.

### Proof Generation (`/prove`)
- **Process**:
    1. Fetches the current `merkle_root` and `merkle_path` from the [[Kingdom_2_Blockchain|Blockchain]].
    2. Uses the `Barretenberg` WASM engine to execute the [[Kingdom_1_Cryptography|Noir Circuit]].
    3. Passes private witnesses from the wallet and public inputs (Scope, Root).
    4. Outputs a ZK Proof string and a Nullifier.

## 🛠️ Key Files
- `src/app/wallet/page.tsx`: UI for managing the local identity.
- `src/app/prove/page.tsx`: UI for generating the ZKP.
- `src/utils/zk-prove.ts`: The bridge between the React frontend and the Noir WASM backend.
